## 1. 模块概述

### 1.1 模块描述
sj-auth模块负责长安行系统的用户认证和授权功能，包括司机登录、Token生成与验证、权限管理等。

### 1.2 模块职责
- 司机登录认证
- JWT Token生成与刷新
- Token验证与解析
- 密码加密与验证
- 司机状态检查

### 1.3 依赖关系
- 依赖：Prisma ORM（数据库操作）
- 被依赖：其他业务模块（需要认证的接口）

## 2. 功能设计

### 2.1 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| SJ-AUTH-001 | 司机登录 | 司机使用手机号和密码登录系统 | 高 |
| SJ-AUTH-002 | Token生成 | 登录成功后生成JWT Token | 高 |
| SJ-AUTH-003 | Token验证 | 验证请求中的Token是否有效 | 高 |
| SJ-AUTH-004 | Token刷新 | 刷新过期前的Token | 中 |
| SJ-AUTH-005 | 司机登出 | 司机退出登录 | 中 |
| SJ-AUTH-006 | 获取当前用户信息 | 获取当前登录司机的信息 | 高 |

### 2.2 功能流程

#### 2.2.1 司机登录流程
```
1. 司机输入手机号和密码
2. 系统验证手机号是否存在
3. 系统验证密码是否正确
4. 系统检查司机状态是否正常
5. 生成JWT Token
6. 返回Token和司机信息
```

#### 2.2.2 Token验证流程
```
1. 从请求头中获取Authorization Token
2. 验证Token格式
3. 验证Token签名和有效期
4. 从Token中解析司机ID
5. 验证司机是否存在且状态正常
6. 返回验证结果
```

## 5. 安全设计

### 5.1 密码安全
- 使用bcrypt进行密码加密
- 密码强度要求：至少6位
- 密码存储：只存储哈希值，不存储明文

### 5.2 Token安全
- 使用JWT (JSON Web Token)
- Token有效期：24小时
- Token刷新：可在过期前1小时内刷新
- Token存储：客户端存储（建议使用HttpOnly Cookie）

### 5.3 接口安全
- 所有需要认证的接口必须携带有效Token
- 登录接口有频率限制（防止暴力破解）
- 敏感操作需要验证用户身份

## 6. 错误码设计

| 错误码 | 错误信息 | 说明 |
|-------|---------|------|
| SJ_AUTH_001 | 手机号或密码错误 | 登录验证失败 |
| SJ_AUTH_002 | 账号已被禁用 | 司机状态异常 |
| SJ_AUTH_003 | Token无效 | Token格式或签名错误 |
| SJ_AUTH_004 | Token已过期 | Token超过有效期 |
| SJ_AUTH_005 | 未登录 | 请求未携带Token |
