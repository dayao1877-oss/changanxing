# sj-daka 打卡管理模块设计文档

---

## 1. 模块概述

### 1.1 功能描述
司机在执行订单过程中，在指定位置打卡（始发地、途径点、目的地），并拍摄仪表盘公里数照片作为验证依据。

### 1.2 技术依赖
- 数据库：sj_daka_jilu表
- 认证：sj-auth JWT Token
- 工具函数：sj-common（sjRequireAuth、sjConvertBigInts）

---

## 2. 数据结构

### 2.1 打卡记录实体

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| sj_id | Int | ✅ | 主键ID |
| sj_dingdan_id | Int | ✅ | 订单ID |
| sj_siji_id | Int | ✅ | 司机ID |
| sj_daka_leixing | Int | ✅ | 打卡类型：1-始发地，2-途径点，3-目的地 |
| sj_tujingdian_xuhao | Int | ❌ | 途径点序号（仅sj_daka_leixing=2时有值） |
| sj_weizhi | String(100) | ❌ | 经纬度坐标，格式："经度,纬度" |
| sj_dizhi | String(200) | ❌ | 详细地址 |
| sj_tupian_lujing | String(200) | ❌ | 公里数照片存储路径（后续支持上传） |
| sj_daka_shijian | DateTime | ✅ | 打卡时间 |
| sj_shangchuan_zhuangtai | Int | ✅ | 上传状态：0-待上传，1-已上传 |

---

## 3. 功能设计

### 3.1 功能列表
1. **提交打卡** - 司机在指定位置打卡
2. **查询打卡记录列表** - 按订单或司机查询打卡记录
3. **查询打卡详情** - 查询单个打卡的详细信息

### 3.2 业务规则
1. 只能对自己的订单打卡
2. 打卡类型必须是1、2、3
3. 始发地打卡后才能进行后续打卡
4. 打卡时间必须晚于订单开始时间
5. 同一订单的打卡顺序必须正确（始发 → 途径 → 目的）

---

## 4. 模块接口

| 接口名 | 方法 | 路径 | 说明 |
|-------|------|-----|------|
| 提交打卡 | POST | /api/sj-daka/tijiao | 司机提交打卡记录 |
| 查询订单的打卡记录 | GET | /api/sj-daka/dingdan/{dingdanId} | 查询某个订单的所有打卡记录 |
| 查询打卡详情 | GET | /api/sj-daka/{id} | 查询单个打卡的详情 |

---

## 5. 响应格式

统一响应格式：

```json
{
  "sj_code": 200,
  "sj_message": "success",
  "sj_data": {}
}
```
