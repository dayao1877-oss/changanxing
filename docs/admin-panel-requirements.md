# 长安行 - 后台管理系统需求文档

> 本文档描述后台管理系统的功能需求和实现方案

---

## 📋 系统概述

**目标用户**：姚师傅、王磊（管理员）

**使用场景**：
- 上传和更换首页背景图
- 修改司机介绍信息
- 更新车辆信息和照片
- 管理订单状态
- 上传摄影作品
- 修改价格和服务说明

**访问方式**：Web 端管理后台（需要登录）

---

## 🔐 管理员登录

### 功能需求

**登录页面**：
- 手机号 + 密码登录
- 或手机号 + 验证码登录
- 记住登录状态（7 天免登录）

**权限管理**：
- 超级管理员：姚师傅（可以管理所有内容）
- 普通管理员：王磊（只能管理自己的车辆和订单）

### 一、网站配置管理

#### 1.1 首页配置 ⭐⭐⭐⭐⭐

**可配置项**：

### 二、司机信息管理

#### 2.1 司机档案 ⭐⭐⭐⭐⭐

**字段**：
```yaml
基本信息:
  - 姓名（文本输入）
  - 头像（图片上传，建议 400x400px）
  - 电话号码（文本输入）
  - 微信号（文本输入）
  - 驾龄（数字输入）
  - 个人介绍（富文本编辑器）

  - 是否在前端显示（开关）
```

**操作**：
- 新增司机
- 编辑信息
- 上传头像
- 启用/禁用
- 删除（逻辑删除）

**查看内容**：
```yaml

  - 订单号

  - 客户姓名、电话、微信
```

## 🛠️ 技术实现方案

### 后端 API 设计

#### RESTful API 接口

```typescript
// 管理员认证
POST   /api/admin/login              // 登录
POST   /api/admin/logout             // 登出
GET    /api/admin/profile            // 获取当前管理员信息

// 网站配置
GET    /api/admin/configs/homepage   // 获取首页配置
PUT    /api/admin/configs/homepage   // 更新首页配置
POST   /api/admin/configs/upload     // 上传图片

// 司机管理
GET    /api/admin/drivers            // 司机列表
POST   /api/admin/drivers            // 创建司机
PUT    /api/admin/drivers/:id        // 更新司机
DELETE /api/admin/drivers/:id        // 删除司机

// 车辆管理
GET    /api/admin/vehicles           // 车辆列表
POST   /api/admin/vehicles           // 创建车辆
PUT    /api/admin/vehicles/:id       // 更新车辆
DELETE /api/admin/vehicles/:id       // 删除车辆

// 档期管理
GET    /api/admin/schedule/:date     // 获取某日期档期
PUT    /api/admin/schedule/:date     // 设置日期状态

// 订单管理
GET    /api/admin/orders             // 订单列表（带筛选）
GET    /api/admin/orders/:id         // 订单详情
PUT    /api/admin/orders/:id         // 更新订单状态

// 作品管理
GET    /api/admin/portfolio          // 作品列表
POST   /api/admin/portfolio          // 上传作品
PUT    /api/admin/portfolio/:id      // 更新作品
DELETE /api/admin/portfolio/:id      // 删除作品

// FAQ 管理
GET    /api/admin/faqs               // FAQ 列表
POST   /api/admin/faqs               // 创建 FAQ
PUT    /api/admin/faqs/:id           // 更新 FAQ

// 数据统计
GET    /api/admin/stats/overview     // 概览数据
GET    /api/admin/stats/trends       // 趋势数据
```

### 数据库表设计

```sql
-- 管理员表
CREATE TABLE admins (
  id BIGINT PRIMARY KEY,
  username VARCHAR(50) UNIQUE,
  phone VARCHAR(20) UNIQUE,
  password_hash VARCHAR(255),
  role ENUM('super_admin', 'admin'),
  created_at TIMESTAMP
);

-- 网站配置表
CREATE TABLE site_configs (
  id BIGINT PRIMARY KEY,
  config_key VARCHAR(100) UNIQUE,
  config_value JSON,
  updated_at TIMESTAMP
);

-- 司机表（已有）
-- 车辆表（已有）
-- 订单表（已有）

-- 摄影作品表
CREATE TABLE portfolio_items (
  id BIGINT PRIMARY KEY,
  title VARCHAR(100),
  images JSON,
  location VARCHAR(100),
  shoot_date DATE,
  description TEXT,
  sort_order INT DEFAULT 0,
  is_visible BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP
);

-- FAQ 表
CREATE TABLE faqs (
  id BIGINT PRIMARY KEY,
  question TEXT,
  answer TEXT,
  category VARCHAR(50),
  sort_order INT DEFAULT 0,
  is_visible BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP
);
```

## 💡 特别说明

### 安全性考虑

1. **权限验证**：
   - 所有后台 API 需要 JWT Token
   - 超级管理员和普通管理员权限分离
   - 敏感操作需要二次确认

2. **数据安全**：
   - 图片上传限制大小和类型
   - 防止 SQL 注入
   - 敏感信息加密存储

3. **操作日志**：
   - 记录所有修改操作
   - 支持操作历史追溯
