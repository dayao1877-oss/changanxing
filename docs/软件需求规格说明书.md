# 司机订单管理系统 - 软件需求规格说明书（SRS）

> 文档版本：v1.0
> 创建日期：2026-04-23
> 维护人：技术负责人

---

## 1. 引言

### 1.1 编写目的
本文档旨在详细描述司机订单管理系统的软件需求，包括功能需求、非功能需求、性能标准、安全要求等，作为后续开发、测试和验收的依据。开发过程将严格按照本文档执行，确保需求落地率100%。

### 1.2 项目背景
老板需要一套完整的司机订单管理系统，用于管理车队司机的订单执行过程。系统需要确保订单完成的真实性，通过公里数照片和定位轨迹作为与甲方结账和司机结算的依据。

### 1.3 项目目标
- 实现订单的创建、分配、执行、完成全流程管理
- 通过公里数照片和定位轨迹验证订单执行的真实性
- 完善的财务管理，支持与甲方和司机的结算
- 实时监控司机位置和订单状态
- 提供紧急情况SOS功能，保障司机安全

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| SRS | 软件需求规格说明书（Software Requirements Specification） |
| PRD | 产品需求文档（Product Requirements Document） |
| Nuxt 3 | Vue全栈框架，用于构建Web后台和后端API一体化应用 |
| sj前缀 | 所有命名的前缀，代表"司机"拼音首字母 |
| SOS | 紧急求助功能 |

### 2.1 系统架构
本系统采用以下架构：

```
┌─────────────────────────────────────────────────────────┐
│                    Android APP端                          │
│                    （司机端）                              │
└──────────────────────────┬──────────────────────────────┘
                           │
                           │ HTTPS
                           │
┌──────────────────────────▼──────────────────────────────┐
│              Nuxt 3 全栈应用一体化                        │
│  ┌─────────────────────────────────────────────────────┐│
│  │              Web后台管理端                            ││
│  │         （管理员/老板使用）                           ││
│  └───────────────────────────┬─────────────────────────┘│
│                              │                         │
│  ┌───────────────────────────▼─────────────────────────┐│
│  │         Nuxt Server Routes API                       ││
│  │         （基于H3框架）                                ││
│  └───────────────────────────┬─────────────────────────┘│
│                              │                         │
└──────────────────────────────┼─────────────────────────┘
                               │
┌──────────────────────────────▼─────────────────────────┐
│                    MySQL 8.0 数据库                     │
└────────────────────────────────────────────────────────┘
```

### 2.2 用户角色

| 角色 | 使用端 | 主要权限 |
|------|--------|----------|
| 系统管理员（老板） | Web后台管理端 | 最高权限，可访问所有功能 |
| 司机 | Android APP端 | 仅能访问自己的订单和数据 |

### 2.3 系统范围

#### 包含功能
- Android司机端APP
- Nuxt 3全栈应用（Web后台管理 + 后端API一体化）
- 数据库存储
- 文件存储服务

#### 不包含功能
- iOS版本（暂不支持）
- 乘客/甲方下单端（订单由老板手动创建或司机自建）
- 第三方支付集成（财务仅做记录，支付线下进行）
- Redis缓存（不使用缓存）

## 4. 非功能需求

### 4.1 性能标准

| 指标 | 要求值 | 说明 |
|------|--------|------|
| API响应时间 | ≤200ms | 所有API接口的响应时间 |
| 并发用户数 | ≥500 | 支持同时在线的用户数 |
| 页面加载时间 | ≤2s | Web后台管理端页面加载 |
| APP启动时间 | ≤3s | Android APP启动时间 |
| 照片上传时间 | ≤10s（WiFi）/ ≤30s（4G） | 公里数照片上传 |
| 系统可用性 | ≥99.5% | 系统在线可用时间占比 |

### 4.2 兼容性要求

#### Android APP兼容性
- 最低版本：Android 6.0（API 23）
- 目标版本：Android 13（API 33）
- 屏幕适配：支持主流分辨率，适配屏幕尺寸1080P+

#### Web后台管理端兼容性
- 浏览器：Chrome 80+、Firefox 75+、Edge 80+
- 分辨率：1920x1080及以上

#### 弱网环境处理
- 弱网环境下能自动降级处理
- 给出友好提示
- 支持本地缓存和断网续传

### 4.3 安全性要求

#### 输入校验
- 对所有用户输入进行严格校验
- 防止SQL注入攻击
- 防止XSS（跨站脚本）攻击
- 防止CSRF（跨站请求伪造）攻击

#### 数据安全
- 密码使用BCrypt加密存储
- 敏感信息（身份证号、手机号等）加密存储
- 数据传输使用HTTPS加密
- Token机制：JWT，有效期7天

### 5.3 API接口设计

#### 接口规范
- 基础URL：生产环境 https://api.yourdomain.com/api/v1，测试环境 http://test-api.yourdomain.com/api/v1
- 接口路径格式：/api/sj/xxx
- 通用响应格式：
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
- 错误码定义：200成功，400请求参数错误，401未授权，403禁止访问，404资源不存在，500服务器错误

- 内部实现细节对外不可见
- 避免循环依赖
